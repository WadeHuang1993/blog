2019 回顧工作與學習歷程

2019 年可以說是既充實又偷懶的一年。為什麼呢？因為花了很多時間在學習理論知識，包含：重構、程式架構、單元測試、領域驅動測試、行為驅動測試 …等等。雖然每個知識看起來像是完全獨立的領域，但對 2019 的我來說，它們都是應付 Legacy Application 的利器。

為什麼又說偷懶？部落格和作品幾乎停擺！理論知識必須經過內化與堆疊，才能在真實專案中實踐。不像學習新技術一樣，可以把應用新技術的步驟依序記錄下來分享到部落格上。加上這一年很榮幸受邀到高雄科技大學替一批學生教導 PHP 式設計課程，備課與上課的時間幾乎吃掉了三個月的休閒時間，也就是平常我能用來自學或練習的時間。

工作上呢？這份工作是我搬來高雄的第一份工作，工作內容主要是維護一個大型的 Legacy Application。
雖然說是維護，其實仍不斷有大量新需求一直進來。這也是為什麼我必須花費大量時間閱讀面對 Legacy Application 相關的書籍和技術的原因。很慶幸我也有機會將學到的知識導入專案中，增加部分功能的彈性；或是阻止部分功能腐敗。這些成就感又將變成讓我繼續學習的動力。

可惜 2019 對我來說是一眨眼就結束，都到年尾了還沒來得及紀錄下學習與實踐的東西。所以想先以流水帳的形式紀錄一下 2019 年所學的知識和工作中引入的實踐。
