軟體開發中的“無緒”

無緒 (Cluelessness) 由 Martin Rinard 提出。他在演講時指出：

在開發和維護軟體系統時，應該避免讓開發人員深入了解系統。

因為人的大腦功能是有限的，只能處理有限的數據。若要建立一個日益變大的應用程式，就必須學習「如何讓每個人都只了解應用程式的某一部份，也能完成軟體開發」。

“無緒” 並不是一個貶義詞。它用來區別兩種層次的理解水平。

  1. 淺層理解：指對事物的了解程度僅限於掌握使用方法即可。
  2. 深層理解：指對掌握了事物背後的原則、規律、原理。

在日常生活中的“無緒”

生活中我們通常只需要 淺層理解。我們不需要理解冰箱如何發冷，就可以利用冰箱來冷藏食品。當然，也有一些人需要了解更深入的內容。例如冰箱的維修人員就需要了解較深入的領域知識。但即便如此，維修人員所需要的知識仍屬於 淺層理解，他們也並不需要了解事物背後的每個小細節和原理。

當然也可以學習每件事物背後的知識原理，但是必要性通常不大。因此大多數人在日常生活中只需要 淺層理解 就足夠了。

軟體開發中的“無緒”

在軟體開發中，“無緒” 也表示大部分的時間中，只需要 淺層理解 就足以應付工作。
這裡的 淺層理解 並不是在說工程師不需要懂得編寫程式。以下舉個例子來說明軟體開發中的 “無緒”。

無緒範例「販售產品的網站」：

我想要實現一個 Side Project，一個販售產品的網站。實現這個網站有兩種做法：

實作方法一：

在專案初始階段，我需要打開 HTTP 協定的文件，解析 HTTP 傳輸格式。除此之外還要研究如何實現 POST、GET 請求…. 閱讀 RFC 文件，並實現文件中的各項內容。全部都搞定後才開始打造我的 Side Project。

實作方法二：

在專案初始階段，在作業系統中下載並安裝 Apache、PHP、MySQL，設定妥當後即開始打造我的 Side Project。

現代軟體都是基於組件組裝出來的。沒有人會獨自從頭到尾完成所有內容。在一個操作系統上安裝 Web 服務器後，開始編寫 HTML 對現在來說是易如反掌。但事實上 Web 服務服務系統複雜到極點，相信沒人敢說自己了解整個 Web 服務系統的所有內容。

軟體開發中的 “無緒” 就是讓 工程師可以在對系統很少了解的前提下，仍然可以完成開發的工作。
